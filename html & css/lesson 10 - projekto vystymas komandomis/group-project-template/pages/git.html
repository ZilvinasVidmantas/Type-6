GIT - projektų versijavimo kontrolė

Projektų versijavimo kontrolė, tai būdas keliems žmonės ar komandoms nepertraukiamai vystyti projektą.

Visi projekto failai, kodas ir kita informacija patalpinama globalioje repozitorijoje, kuria gali naudotis visi projekto vystytojai. Globalios repozitorijos kodas nėra keičiamas tiesiogiai. Klaidų prevencijos ir nenutraukiamui darbui reikalinga tvarka, kuri bus aprašoma sekančiose potemėje.

Pasirengimas darbui

  • Susikurti repozitoriją serveriuojančios kompanijos paskyrą, mūsų atveju Github.
  • Parsisiųsti programą, kuri mums suteikia sąsaja vykdyti GIT komandas - GIT bash.
  • Atsidarius terminalą, reikia užregistruoti kas jūs esate, jog tai matytūsi jūsų kodo pakitimuose, tam reikalinga įvykdyti šias komandas:
    • git config --global user.name "VARDAS PAVARDĖ"
    • git config --global user.email "JŪSŲ_PAŠTAS@domenas.com"
  • Reikia pasiklonuoti jus dominančia repozitoriją, pavyzdžiui ZilvinasVidmantas/TYPE2
    • git clone https://github.com/ZilvinasVidmantas/TYPE2.git

Failų atnaujinimas/atsisiuntimas

Keičiant failus savo kompiuteryje, reikia užfiksuoti pakitimus naudojant komandą: 

git add .
Tuomet užfiksavus, reikia juos patvirtinti su patvirtinimo žinute: 
git commit -m "atliktą darbą apibudinanti žinutė"
Tokiu būdu yra užtvirtinamas pakitimas jūsų darbinėje šakoje privačiai. Tam kad šiuos pakitimus pamatytų kiti žmonės reikia juos paviešinti. Kitaip tariant savo pakitimus nusiųsti į globalią repozitoriją, tai darome rašydami: 
git push
Norint atnaujinti privačios repozitorijos informaciją, turi nebūti jokių pakitimų kuomet bandysite tai daryti. Prieš parsisiunčiant informaciją reikėtų užtvirtinti visus pakitimus (git commit ...). Failų atsiuntimas daromas rašant tokią komandą: 
git pull

Naujo funkcionalumo kūrimas

Tam kad dirbant komandoje galėtumėte dirbti nepertraukiamai laikomasi sutartinės tvarkos.
Ši tvarka pateikta iliustracija žemiau.
